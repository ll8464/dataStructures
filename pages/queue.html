Queue

First-in-First-Out (FIFO)

The queue data structure is linear and allows for operations to be performed in First-in-First-Out order. A way to visualize it is waiting in line for a movie. Those who get there first will be the first ones to get served. All who comes after will be get their tickets (be processed) after, in the order they arrived. The process of adding elements to an queue is called enqueue and is done from the tail/rear of the structure. Removing elements occures at the head/front of the structure in a process called dequeue. Must know algorithms involving queues include Breadth First Search (BFS) and Binary Tree Level Order Traversal.

Return true if the operation is successful. */ public boolean enQueue(int x) { data.add(x); return true; }; /** Delete an element from the queue. Return true if the operation is successful. */ public boolean deQueue() { if (isEmpty() == true) { return false; } p_start++; return true; } /** Get the front item from the queue. */ public int Front() { return data.get(p_start); } /** Checks whether the queue is empty or not. */ public boolean isEmpty() { return p_start >= data.size(); } }; public class Main { public static void main(String[] args) { MyQueue q = new MyQueue(); q.enQueue(5); q.enQueue(3); if (q.isEmpty() == false) { System.out.println(q.Front()); } q.deQueue(); if (q.isEmpty() == false) { System.out.println(q.Front()); } q.deQueue(); if (q.isEmpty() == false) { System.out.println(q.Front()); } } }
